# Game Design Overview

## Vision
Grotto Quest is a 3D action dungeon crawler centered on build expression, party tactics, and procedural replayability.

## Design Pillars
- Meaningful player choice in character and build setup.
- Replayability through encounter and dungeon variation.
- Learn-by-doing onboarding (minimal hard tutorial walls).
- Reward adaptation and decision quality over repetitive play.

## Core Loop
1. Enter dungeon.
2. Fight and explore.
3. Collect loot and progression resources.
4. Defeat encounter climax/boss.
5. Return to hub for preparation and upgrades.
6. Re-enter with deeper challenge.

## Combat Direction
- Real-time, high-density action pacing.
- Movement, timing, and skill sequencing matter.
- Party members can be directed or partially autonomous.
- Enemy telegraphs and encounter patterns should demand response, not static rotations.

## Progression Direction
- Early complexity is controlled; depth scales over time.
- Build identity comes from skills, traits, gear, and modifiers.
- Endgame systems should be introduced gradually, not gated behind complete campaign progress.

## MVP Scope
- Character creation basics.
- Core attack/dodge loop with small starter skill set.
- Procedural dungeon slice (3-5 rooms + mini-boss).
- Basic loot and leveling.
- Minimal hub services (shop/storage/prep).

## Long-Term Direction
- Expanded class/skill variety.
- Deeper party command systems.
- Stronger meta progression and replay incentives.
- Broader biome/enemy/boss content.

# WR-172 Architecture Behavior Truth And Extension Readiness Closure Plan

Executable authority lives in `plan.contract.yaml`; this prose is a human-readable companion.

## Goal

Close the final UiProgram architecture truth gap by making semantic conformance behavior-first. The track must prove accepted architecture requirements, extension readiness, retained compatibility, and durable completion without treating future product features as part of this architecture track.

## Boundaries

- Do not start MaterialProgram implementation.
- Do not extract `foundation/meta`.
- Do not create placeholder folders.
- Do not add unrelated UI product features.
- Do not satisfy strong truth from generated reports, closeouts, certificates, or source-symbol checks alone.

## Required Outcome

- Every semantic conformance check has behavior probe IDs and probe files.
- Source-symbol checks remain shape guards only.
- Future product features are either accepted architecture requirements, explicitly deferred with authority, or registered for separate future tracks.
- `task track` reports truth blockers while strong claims are blocked.
- Fresh truth certificates may become authoritative only after behavior-probe-backed verification passes with zero findings, gaps, risks, and drift.

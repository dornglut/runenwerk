# PM-EDITOR-UX-003 Layered Editor Design System Migration Closeout

## Summary

`PM-EDITOR-UX-003` / `WR-113` completes the first runtime-proven editor
design-system migration path. Editor theme documents now expose typed generic
`ui_theme` token graph declarations with provenance, the editor Story Lab owns
a primary-button product recipe/state evidence contract over generic
`ui_definition` recipes, and the app-owned Editor UX runner records
token/recipe/state evidence in the manifest.

This closeout does not claim final shell polish, standalone UI Designer
workbench completion, graph productization, all-surface migration, game UI
readiness, or final no-gap certification. Those remain visible in
`PM-EDITOR-UX-004` through `PM-EDITOR-UX-009`.

## Implementation Summary

- `domain/editor/editor_definition/src/theme.rs` module `theme` adds
  `EDITOR_THEME_TARGET_PROFILE` and `editor_theme_token_graph`, mapping
  `EditorThemeDefinition` color, spacing, radius, and typography packages into
  generic `ThemeTokenGraph` / `ThemeTokenDeclaration` values.
- `domain/editor/editor_definition/src/theme.rs` function `form_theme_tokens`
  now forms `ThemeTokens` through `resolve_theme_tokens` and
  `ThemeTokenResolveRequest::activate("editor.workbench")`, preserving existing
  editor theme diagnostics and persisted schema version 1 compatibility.
- `domain/editor/editor_shell/src/story_lab/design_system.rs` module
  `design_system` adds `EditorUxDesignSystemEvidence`,
  `editor_design_system_recipe_library`, `primary_button_recipe`, and
  `primary_button_design_system_evidence` for the migrated primary-button
  product pattern.
- `domain/editor/editor_shell/src/story_lab/story.rs` struct `EditorUxStory`
  adds optional `design_system_evidence` and
  `with_design_system_evidence`, keeping evidence as editor product metadata
  instead of app-owned style truth.
- `domain/editor/editor_shell/src/story_lab/scenario.rs` struct
  `EditorUxScenarioMatrix` adds `design_system_state_variants` so migrated
  stories name density, high-contrast, reduced-motion, disabled, focused,
  warning, error, overflow, and long-label state coverage.
- `domain/editor/editor_shell/src/story_lab/catalog.rs` function
  `default_editor_ux_stories` attaches primary-button token, recipe, and state
  evidence to the existing primitive button story.
- `apps/runenwerk_editor/src/shell/editor_lab_evidence/mod.rs` enum
  `EditorLabEvidenceArtifactKind` adds `DesignSystemReport` as the app-owned
  artifact kind for token/recipe/state manifest proof.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/manifest.rs` struct
  `EditorUxEvidenceRun` adds typed `design_system_evidence` entries and
  `EditorUxEvidenceManifest::validate` now rejects migrated stories without a
  design-system artifact and matching token/recipe/state evidence.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/runner.rs` function
  `run_story` writes design-system evidence for migrated stories and keeps
  native screenshot absence represented as a typed platform-impossible report.

## Drift Check

The implementation matches the WR-113 contract:

- Generic token and recipe contracts remain in `domain/ui`.
- Editor product token adapters, recipe evidence, and Story Lab state metadata
  remain in `domain/editor`.
- App-owned native evidence remains in `apps/runenwerk_editor` and is not
  treated as source truth.
- The migrated primary-button path proves the source-to-runtime chain from
  editor theme package, through `ThemeTokenGraph` and `UiRecipeLibrary`, into
  Story Lab state metadata, and finally into app-owned manifest evidence.
- Future `game.runtime` UI remains separated; this closeout uses target-profile
  ids and does not make game UI depend on editor shell vocabulary.

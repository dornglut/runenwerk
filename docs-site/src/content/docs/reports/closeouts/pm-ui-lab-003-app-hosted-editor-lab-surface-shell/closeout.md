# PM-UI-LAB-003 App-Hosted Editor Lab Surface Shell Closeout

## Scope

`WR-095` completes the bounded `PM-UI-LAB-003` implementation slice: the
Editor Design workspace now presents app-hosted Editor Lab surfaces through
typed hierarchy, palette, retained preview or degraded canvas, inspector,
review, diagnostics, and preview-console view models instead of normal
debug-style self-authoring action lists.

This slice productizes the shell and direct-control routing only. It does not
complete operation history, deterministic visual operation diffs, project IO,
full diff/apply/rollback, screenshot matrices, accessibility checks,
performance evidence, public API examples, or no-gap perfectionist
certification. Those remain downstream `PM-UI-LAB-004` through
`PM-UI-LAB-007` or later audit scope.

## Implementation Summary

- `domain/editor/editor_shell/src/surfaces/editor_definition.rs` module
  `editor_definition` now defines app-neutral `EditorLabSurfaceViewModel`
  contracts and typed sub-view models for definition hierarchy, palette,
  canvas preview, inspector fields, review rows, diagnostics, console output,
  direct actions, text fields, and degraded states.
- `domain/editor/editor_shell/src/composition/build_editor_lab_surface.rs`
  module `build_editor_lab_surface` composes those view models into retained UI
  controls with deterministic `SurfaceWidgetScope` widget ids and route tables
  for enabled `SurfaceLocalAction::EditorDefinition` actions.
- `domain/editor/editor_shell/src/commands/map_interactions.rs` function
  `map_interaction_to_shell_command` maps retained text-input interactions for
  rename, UI node text, theme color, and workspace tab label edits into typed
  editor-definition surface actions. It does not mutate app state or move
  provider behavior into a domain crate.
- `apps/runenwerk_editor/src/shell/providers/self_authoring.rs` module
  `self_authoring` translates `SelfAuthoringWorkspaceState` into typed Editor
  Lab view models for supported Editor Design surfaces. `UiCanvas` still uses
  retained `ui_definition` previews for previewable UI documents, while
  non-previewable documents render typed recovery/degraded state.
- `apps/runenwerk_editor/src/shell/providers/mod.rs` module `providers` wires
  the Editor Lab builder into the app-owned provider boundary while preserving
  the PM-002 registry-backed surface/provider path.
- `apps/runenwerk_editor/src/shell/tests.rs` test
  `pm_ui_lab_003_runtime_evidence_reports_app_hosted_editor_lab_shell` proves
  the app-hosted shell chain, direct edit dispatch, retained preview update,
  degraded canvas state, diagnostics/review surfaces, and preview-console
  feedback.

## Acceptance Evidence

- Normal supported Editor Design hierarchy, palette, inspector, review,
  diagnostics, and console surfaces are built from typed Editor Lab view models
  in `apps/runenwerk_editor/src/shell/providers/self_authoring.rs`, not from
  hard-coded line/action panel workflows.
- `UiCanvas` preserves deterministic retained `ui_definition` preview output
  for previewable UI documents and renders typed degraded recovery state for
  non-previewable documents.
- Direct controls route through `SurfaceLocalAction::EditorDefinition`, then
  app-owned `ShellCommand` dispatch, before mutating
  `SelfAuthoringWorkspaceState`.
- `pm_ui_lab_003_runtime_evidence_reports_app_hosted_editor_lab_shell` proves a
  supported edit updates app-owned draft state and retained preview content.
- `editor_lab_provider_builds_typed_direct_control_surfaces` proves supported
  app provider surfaces expose typed Editor Lab controls and do not include the
  old hard-coded demo edit strings.
- `editor_lab_canvas_degraded_state_is_typed_and_recoverable` proves
  non-previewable document state is visible as typed degraded UI with recovery
  actions.
- `editor_lab_surface_routes_actions_and_text_fields` and
  `editor_lab_surface_keeps_disabled_actions_out_of_routes` prove deterministic
  composition routing and disabled-action behavior.
- `editor_lab_text_input_maps_to_typed_editor_definition_actions` proves
  retained text-input interactions route to typed editor-definition actions.

## Drift Check

The phase completion drift check found the implementation aligned with the
accepted PM-003 shell design and WR-095 contract:

- Generic `domain/ui/ui_definition` remains behavior-free and only provides
  retained preview formation for authored UI definitions.
- App provider behavior, runtime dispatch, console feedback, and draft-state
  mutation remain in `apps/runenwerk_editor`.
- `domain/editor/editor_shell` owns app-neutral view-model and retained
  composition contracts without owning project IO, runtime activation, or app
  mutation authority.
- Registry-backed surface metadata from PM-002 remains the normal mounted
  surface path; WR-095 does not introduce a parallel surface identity source.
- Later productization slices remain incomplete and must not be inferred from
  the PM-003 shell proof.

# PM-UI-LAB-005 Persistence Project IO Diff Apply And Rollback Closeout

## Scope

`WR-086` completes the bounded `PM-UI-LAB-005` implementation slice: Editor Lab
drafts now have an app-owned project package/store boundary, deterministic
apply review, reject/apply decisions, typed activation reports, failed
activation input preservation, reload-last-applied behavior, and
snapshot-backed rollback.

This slice does not implement PM-006 screenshot matrices, visual diffing,
accessibility checks, performance evidence, broad degraded-provider scenario
coverage, PM-007 public API docs/examples/final closeout, game-runtime UI
projection execution, or no-gap perfectionist certification.

## Implementation Summary

- `apps/runenwerk_editor/src/shell/editor_lab_project/mod.rs` module
  `editor_lab_project` defines the app-owned `EditorLabProjectPackage`,
  `EditorLabDocumentStore`, `DefinitionApplyReview`, activation report,
  pending activation, project load/store/import, and rollback record
  contracts.
- `apps/runenwerk_editor/src/shell/self_authoring.rs` module
  `self_authoring` creates deterministic project packages from drafts,
  applied snapshots, and last-applied snapshots; preserves invalid package
  input with typed diagnostics; imports/exports selected definition packages;
  requires `DefinitionApplyReview` before normal apply; records rejected and
  accepted review state; supports reload-last-applied; and records
  snapshot-backed rollback.
- `domain/editor/editor_shell/src/surfaces/editor_definition.rs` module
  `editor_definition` adds typed Editor Lab actions for save package, reload
  package, build apply review, reject review, reload last applied, apply, and
  rollback.
- `apps/runenwerk_editor/src/shell/providers/self_authoring.rs` module
  `self_authoring` exposes package, review, activation report, diagnostics,
  reload, and rollback state through the Command Diff/Preview Console surface
  using typed view models instead of raw debug action lists.
- `apps/runenwerk_editor/src/shell/dispatch_shell_command.rs` function
  `dispatch_shell_command_with_viewport_commands` routes the new Editor Lab
  commands through app-owned project package and review methods, then queues
  accepted definitions for activation with review identity preserved.
- `apps/runenwerk_editor/src/editor_app/state.rs` module `state` replaces
  queue-only activation evidence with typed pending activation requests,
  activation reports, and failed activation preservation.
- `apps/runenwerk_editor/src/runtime/resources.rs` method
  `EditorHostResource::apply_pending_editor_definition_activations` records
  typed activation reports for success, no-live-activation, validation or
  installer failure, and previous-state-preserved failure paths while keeping
  console lines as projections only.

The runtime proof opens the Editor Design profile through the app shell and
proves:

- an Editor Lab project package is saved and reloaded into drafts without live
  runtime activation;
- selected definition export/import uses a typed versioned package;
- invalid project package input is preserved with typed diagnostics;
- selected draft versus applied state creates deterministic apply-review diff
  rows;
- rejected apply preserves draft and applied state;
- accepted apply records a `DefinitionApplyReview`, queues activation, executes
  activation, and records a typed activation report linked to the review;
- failed editor bindings activation preserves the failed input and the previous
  active bindings state;
- reload-last-applied restores the prior applied snapshot over a dirty draft;
- rollback is snapshot-backed and records a typed rollback record;
- the Command Diff surface exposes package, review, activation diagnostic, and
  rollback state through typed Editor Lab view models.

`review-surface-debug.txt` is the PM-005 equivalent visual artifact for this
headless runtime proof. Native screenshots, visual-diff comparisons,
accessibility checks, performance evidence, and broad scenario matrices remain
explicit `PM-UI-LAB-006` scope.

## Acceptance Evidence

- Project IO is owned under `apps/runenwerk_editor/src/shell`, not
  `domain/ui/ui_definition`.
- `domain/ui/ui_definition` remains behavior-free and did not gain project IO,
  editor activation, provider sessions, failed activation preservation, or
  rollback behavior.
- `DefinitionApplyReview` is the normal selected-draft-to-apply path and
  records source snapshots, proposed applied state, deterministic diff rows,
  diagnostics, and decision state.
- Apply and reject preserve source truth: reject does not mutate applied state,
  apply records last-applied and rollback snapshots before queuing activation.
- Runtime activation reports are structured app-owned records, and console
  lines remain only a projection.
- Failed activation preserves failed input and previous active state.
- Rollback restores the recorded prior applied state or recorded no-applied
  state and fails closed when no rollback record exists.
- Runtime evidence proves save, reload, import/export, invalid package
  preservation, diff, reject, apply, activation report, failed activation
  preservation, reload-last-applied, rollback, and provider surface exposure.

# PM-UI-LAB-PERF-005 Persistence Diff Apply API And Examples Ergonomics Closeout

## Scope

`WR-109` completes the bounded `PM-UI-LAB-PERF-005` implementation slice:
normal Editor Lab users can persist, reload, review structural draft-versus-
applied diffs, reject, apply, activate, reload last applied, rollback, and use
the documented public `ui_definition` and `editor_definition` workflows without
private or app-internal shortcuts.

This closeout does not claim final no-gap certification. `PM-UI-LAB-PERF-006`
owns the final audit and any `perfectionist_verified` claim.

## Implementation Summary

- `apps/runenwerk_editor/src/shell/editor_lab_project/mod.rs` module
  `editor_lab_project` extends `DefinitionApplyDiffRow` with typed
  `DefinitionApplyDiffFamily`, `DefinitionApplyDiffKind`, and `summary`
  fields while preserving existing `path`, `before`, and `after` text for
  compatibility.
- `apps/runenwerk_editor/src/shell/self_authoring.rs` function
  `definition_apply_diff_rows` now delegates content changes to structural
  diff helpers instead of emitting one coarse serialized `document.content`
  row.
- `apps/runenwerk_editor/src/shell/self_authoring.rs` functions
  `definition_content_diff_rows`, `ui_template_diff_rows`, and
  `ui_node_diff_rows` emit typed UI-template node paths such as
  `document.content.ui_template.root.<node>.label` for draft-versus-applied
  review.
- `apps/runenwerk_editor/src/shell/providers/self_authoring.rs` function
  `command_review_view_model` renders apply diff family, kind, path, before,
  after, and summary in the Command Diff surface.
- `apps/runenwerk_editor/src/shell/tests.rs` test
  `pm_ui_lab_perf_005_persistence_api_examples_structural_workflow` proves the
  app-owned normal workflow: project package save/reload, invalid package
  preservation, structural apply review, reject, apply, activation report,
  reload-last-applied, rollback, and artifact generation.
- `docs-site/src/content/docs/domain/ui/ui-definition-usage.md` and
  `docs-site/src/content/docs/domain/editor/editor-definition-usage.md` now
  document the public prelude workflow and the behavior-free domain/app-owned
  apply-review boundary in the same terms proven by runtime evidence.
- `docs-site/src/content/docs/reports/implementation-plans/wr-109-ui-lab-persistence-api-examples-ergonomics-closure/plan.md`
  and the WR-109 roadmap row include public example runs as validation gates.

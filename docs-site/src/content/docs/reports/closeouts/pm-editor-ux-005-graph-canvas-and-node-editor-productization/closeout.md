# PM-EDITOR-UX-005 Graph Canvas And Node Editor Productization Closeout

> Terminology note: Current name: UX Lab Scenarios; historical name: Story Lab.

## Summary

`PM-EDITOR-UX-005` / `WR-115` completes a runtime-proven graph canvas product
slice. Material Graph Canvas now has a first-class Editor UX Story Lab product
story with graph interaction, route, readiness, retained UI, focus,
accessibility, diagnostics, timing, and native or platform-impossible evidence.
The generic graph substrate remains domain-neutral, editor shell owns product
story/readiness contracts, and `apps/runenwerk_editor` owns native evidence
execution and manifest validation.

Unfinished SDF, procgen, gameplay, particle, and animation graph families remain
explicitly hidden until productized, so text/action or provider-line fallbacks
cannot masquerade as product graph canvases.

This closeout does not claim shell-wide pattern polish, all registered visible
surface migration, game UI readiness, or final no-gap certification. Those
remain visible in `PM-EDITOR-UX-006` through `PM-EDITOR-UX-009`.

## Implementation Summary

- `domain/editor/editor_shell/src/story_lab/graph_canvas.rs` module
  `graph_canvas` adds `EditorUxGraphCanvasEvidence`,
  `MATERIAL_GRAPH_CANVAS_STORY_ID`,
  `material_graph_canvas_fixture_view_model`,
  `material_graph_canvas_evidence`, and graph scenario state requirements.
- `domain/editor/editor_shell/src/story_lab/story.rs` struct `EditorUxStory`
  adds optional `graph_canvas_evidence` and
  `with_graph_canvas_evidence`, keeping graph proof metadata in editor product
  Story Lab contracts.
- `domain/editor/editor_shell/src/story_lab/catalog.rs` function
  `default_editor_ux_stories` registers the Material Graph Canvas product story
  through `build_material_graph_surface`.
- `domain/editor/editor_shell/src/composition/mod.rs` module `composition`
  re-exports `MATERIAL_GRAPH_CANVAS_WIDGET_ID` so Story Lab scenarios can target
  the product graph canvas without duplicating widget IDs.
- `domain/editor/editor_shell/src/workspace/surface_contract.rs` function
  `tool_surface_readiness_for_definition_id` remains the hide-or-certify gate;
  tests now assert material graph is product while SDF, procgen, gameplay,
  particle, and animation graph canvases stay hidden until productized.
- `apps/runenwerk_editor/src/shell/editor_lab_evidence/mod.rs` enum
  `EditorLabEvidenceArtifactKind` adds `GraphCanvasReport`.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/manifest.rs` struct
  `EditorUxEvidenceRun` adds typed `graph_canvas_evidence`, and
  `EditorUxEvidenceManifest::validate` rejects graph product stories without
  `GraphCanvasReport`, focus traversal, accessibility, diagnostics, timing,
  readiness decisions, interaction/route evidence, and native or
  platform-impossible proof.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/runner.rs` function
  `run_story` emits graph canvas artifacts and PM-EDITOR-UX-005 runtime proof.

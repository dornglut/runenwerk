# PM-EDITOR-UX-008 Game UI Readiness Seam Closeout

## Summary

`PM-EDITOR-UX-008` / `WR-118` completes the bounded game UI readiness seam.
Generic UI validators now prove `game.runtime` target-profile compatibility
for preview matrices, production readiness evidence, view-model binding,
recipes, persistence activation, and visual layout edits. Editor Story Lab has
an editor-owned compatibility evidence report that records the generic checks
without becoming game-runtime UI source truth.

This closeout does not implement game HUD behavior. `PT-GAME-RUNTIME-UI`
remains the owner of runtime HUD implementation, and `PM-EDITOR-UX-009` remains
the final editor UX local-native no-gap certification milestone.

## Implementation Summary

- `domain/ui/ui_definition/src/preview_fixture/mod.rs` module
  `preview_fixture` extends `UiPreviewMatrixAxisKind` with safe area, platform
  prompt, and view-model freshness axes, and `validate_preview_fixtures` rejects
  incomplete `game.runtime` matrix axis coverage.
- `domain/ui/ui_definition/src/production_readiness/mod.rs` module
  `production_readiness` adds `UiReadinessCompatibilityAxis`,
  `game_runtime_required_compatibility_axes`, and validation for
  `game.runtime` evidence packets and inspection reports.
- `domain/ui/ui_definition/src/view_binding/mod.rs` module `view_binding`
  keeps view-model packages fail-closed for missing/stale runtime packets and
  rejects `UiIntentDescriptorRef::EditorCommand` under `game.runtime`.
- `domain/ui/ui_definition/src/component_recipe/mod.rs` module
  `component_recipe` proves `expand_ui_recipe` preserves accessibility, layout,
  and focus contracts for `game.runtime`.
- `domain/ui/ui_definition/src/persistence_activation/mod.rs` module
  `persistence_activation` proves runtime activation still requires migration
  reports, deterministic diffs, and activation diffs.
- `domain/ui/ui_definition/src/visual_layout/apply.rs` module
  `visual_layout::apply` proves visual-layout edits are target-profile gated
  for `game.runtime`.
- `domain/editor/editor_shell/src/story_lab/game_ui_readiness.rs` module
  `game_ui_readiness` adds `EditorUxGameUiReadinessEvidence` and
  `game_ui_readiness_evidence` as editor-owned Story Lab compatibility proof.
- `domain/editor/editor_shell/src/story_lab/mod.rs` module `story_lab` exports
  the game UI readiness evidence module.

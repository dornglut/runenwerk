# PM-EDITOR-UX-004 Standalone UI Designer Workbench Closeout

> Terminology note: Current name: UX Lab Scenarios; historical name: Story Lab.

## Summary

`PM-EDITOR-UX-004` / `WR-114` completes the first runtime-proven standalone UI
Designer workbench path. The editor shell now has first-class workbench pane,
readiness, and route view models; the UI canvas stable key renders a standalone
workbench instead of the old text/action control panel path; the Editor UX Story
Lab registers a standalone workbench story with scenario/state coverage; and
the app-owned manifest rejects workbench claims unless pane, route, readiness,
focus, accessibility, diagnostics, timing, retained UI, and native or
platform-impossible proof are present.

This closeout does not claim graph canvas/node editor productization,
shell-wide pattern polish, all registered visible surface migration, game UI
readiness, or final no-gap certification. Those remain visible in
`PM-EDITOR-UX-005` through `PM-EDITOR-UX-009`.

## Implementation Summary

- `domain/editor/editor_shell/src/surfaces/editor_definition.rs` module
  `editor_definition` adds `UiDesignerWorkbenchViewModel`,
  `UiDesignerWorkbenchPaneViewModel`,
  `UiDesignerWorkbenchReadinessViewModel`, pane kinds, and readiness status
  contracts.
- `domain/editor/editor_shell/src/composition/build_editor_lab_surface.rs`
  function `build_editor_lab_surface` now composes
  `EditorLabSurfaceViewModel::UiDesignerWorkbench` with product-grade workbench
  labeling, panes, route buttons, diagnostics, and readiness evidence.
- `domain/editor/editor_shell/src/story_lab/workbench.rs` module `workbench`
  adds `UI_DESIGNER_WORKBENCH_STORY_ID`,
  `ui_designer_workbench_fixture_view_model`,
  `ui_designer_workbench_evidence`, and required scenario states for the
  standalone workbench proof.
- `domain/editor/editor_shell/src/story_lab/story.rs` struct `EditorUxStory`
  adds optional `workbench_evidence` and `with_workbench_evidence`, keeping the
  proof contract in editor product metadata instead of app state.
- `domain/editor/editor_shell/src/story_lab/catalog.rs` function
  `default_editor_ux_stories` registers the standalone workbench host scenario
  and builds its retained UI through `build_editor_lab_surface`.
- `apps/runenwerk_editor/src/shell/providers/self_authoring.rs` module
  `self_authoring` keeps compatibility provider ownership but routes
  `ToolSurfaceKind::UiCanvas` through `ui_designer_workbench_view_model` so the
  normal UI canvas stable key shows the standalone workbench path.
- `apps/runenwerk_editor/src/shell/editor_lab_evidence/mod.rs` enum
  `EditorLabEvidenceArtifactKind` adds `WorkbenchReport` as the app-owned
  artifact kind for standalone workbench proof.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/manifest.rs` struct
  `EditorUxEvidenceRun` adds typed `workbench_evidence`, and
  `EditorUxEvidenceManifest::validate` now rejects standalone workbench stories
  without `WorkbenchReport`, focus traversal, accessibility, diagnostics,
  timing, pane/route/readiness evidence, and native or platform-impossible
  proof.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/runner.rs` function
  `run_story` writes the standalone workbench evidence artifacts and preserves
  native screenshot absence as a typed platform-impossible report.

  readiness evidence remain in `domain/editor`.

  `apps/runenwerk_editor` and are not treated as canonical UI Designer truth.

  adapters.

  `editor.workbench` target profile and evidence descriptor seams without
  making game UI depend on editor shell vocabulary.

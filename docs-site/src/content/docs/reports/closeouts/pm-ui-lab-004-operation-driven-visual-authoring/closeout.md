# PM-UI-LAB-004 Operation-Driven Visual Authoring Closeout

## Scope

`WR-085` completes the bounded `PM-UI-LAB-004` implementation slice: supported
Editor Lab authoring edits now route through typed `EditorLabOperation`
contracts with deterministic diffs, validation diagnostics, app-owned
operation history, undo/redo, and retained preview or review-surface refresh.

This slice does not implement project IO, package save/load/import/export,
persisted operation logs, complete user-facing diff/apply/reject/reload,
failed activation preservation, rollback productization, preview scenario
matrices, public API examples, game-runtime UI projection, or no-gap
perfectionist certification. Those remain downstream `PM-UI-LAB-005` through
`PM-UI-LAB-007` or later audit scope.

## Implementation Summary

- `domain/editor/editor_definition/src/operation.rs` module `operation` defines
  `EditorLabOperation`, operation kinds, operation reports, deterministic
  operation diffs, rejection diagnostics, and editor-definition reducers for UI
  text, document rename, theme color, workspace layout tab, split, close-tab,
  and generic `ui_definition` visual layout operations.
- `domain/editor/editor_definition/src/lib.rs` module `editor_definition`
  exports the operation contracts as normal editor-definition workflow APIs.
- `apps/runenwerk_editor/src/shell/self_authoring.rs` module
  `self_authoring` owns session-local operation dispatch, accepted-operation
  history, rejected-operation preservation, undo, redo, operation reports, and
  retained preview selection refresh.
- `domain/editor/editor_shell/src/commands/shell_command.rs` module
  `shell_command` adds explicit Editor Lab operation, undo, and redo shell
  commands while retaining compatibility commands for existing direct controls.
- `apps/runenwerk_editor/src/shell/dispatch_shell_command.rs` function
  `dispatch_shell_command_with_viewport_commands` routes supported PM-003
  direct edit commands into `EditorLabOperation` dispatch before app draft
  mutation.
- `domain/editor/editor_shell/src/surfaces/editor_definition.rs` module
  `editor_definition` adds typed undo/redo surface actions, and
  `apps/runenwerk_editor/src/shell/providers/self_authoring.rs` projects
  operation history, last operation status, deterministic diffs, diagnostics,
  and undo/redo availability into the Command Diff surface.
- `apps/runenwerk_editor/src/shell/tests.rs` tests
  `editor_lab_operation_dispatch_records_history_and_refreshes_preview` and
  `pm_ui_lab_004_runtime_evidence_reports_operation_driven_visual_authoring`
  prove accepted edits, rejected diagnostics, history exclusion for rejected
  edits, undo/redo, retained preview refresh, and generic visual layout
  operation reuse.

## Runtime Evidence

Runtime proof artifacts:

- `docs-site/src/content/docs/reports/closeouts/pm-ui-lab-004-operation-driven-visual-authoring/artifacts/runtime-proof.txt`
- `docs-site/src/content/docs/reports/closeouts/pm-ui-lab-004-operation-driven-visual-authoring/artifacts/operation-surface-debug.txt`

The artifacts are generated by:

```text
$env:RUNENWERK_WRITE_PM_UI_LAB_004_EVIDENCE='1'; cargo test -p runenwerk_editor pm_ui_lab_004_runtime_evidence_reports_operation_driven_visual_authoring -- --nocapture
```

The runtime proof opens the Editor Design profile through the app shell and
proves:

- an inspector text edit creates an accepted `EditorLabOperation` and updates
  retained preview text;
- an invalid theme color edit creates a rejected `EditorLabOperation`, surfaces
  typed diagnostics, preserves draft state, and does not enter history;
- undo and redo restore prior and later draft state and refresh retained
  preview output;
- a generic UI visual layout operation uses the
  `domain/ui/ui_definition/src/visual_layout` operation path and produces a
  `UiVisualLayout` operation diff;
- the Command Diff surface shows operation history, last operation status,
  deterministic diff rows, diagnostics, and undo/redo availability.

`operation-surface-debug.txt` is the PM-004 equivalent visual artifact for this
headless runtime proof. Native screenshots, visual-diff comparisons,
accessibility checks, performance evidence, and broader scenario matrices
remain explicit `PM-UI-LAB-006` scope.

## Acceptance Evidence

- Generic visual layout edits reuse `ui_definition::apply_visual_layout_operation`
  and convert its diffs into Editor Lab operation diffs without moving editor
  behavior into `domain/ui/ui_definition`.
- Editor-specific UI text, rename, theme, and workspace layout edits are
  reduced at editor/app-owned boundaries and validated with
  `validate_editor_definition_document`.
- Rejected operations return typed diagnostics, preserve the current draft, and
  do not enter app-owned operation history.
- Accepted operations enter session-local history with before/after document
  state and support undo/redo without reusing scene-runtime history as Editor
  Lab source truth.
- The Command Diff surface projects operation reports, diff rows, diagnostics,
  and undo/redo state through typed Editor Lab view models.
- Runtime evidence proves retained preview refresh after accepted, undo, and
  redo paths.

## Completion Quality

Completion quality is `runtime_proven`.

Known quality gaps:

- PM-004 proves operation routing, deterministic diffs, diagnostics, app-owned
  session history, undo/redo, and retained preview refresh. It does not prove
  persisted operation logs or project/package IO.
- Project IO, package save/load/import/export, migration, full user-facing
  diff/apply/reject/reload, failed activation preservation, and rollback remain
  `PM-UI-LAB-005`.

  `PM-UI-LAB-006`.

  remains later `PT-UI-LAB-PERFECTION` or equivalent scope.

## Drift Check

The phase completion drift check found the implementation aligned with the
accepted PM-004 operation design and WR-085 contract:

- Generic `domain/ui/ui_definition` still owns only behavior-free generic UI
  visual layout operation mechanics.
- Editor-definition-specific operation contracts and reducers are owned by
  `domain/editor/editor_definition`.
- App-owned draft mutation, operation history, undo/redo, console feedback, and
  runtime evidence remain in `apps/runenwerk_editor`.
- `domain/editor/editor_shell` owns typed surface action and review view-model
  contracts without owning app mutation or project IO.
- Later productization slices remain incomplete and must not be inferred from
  the PM-004 operation proof.

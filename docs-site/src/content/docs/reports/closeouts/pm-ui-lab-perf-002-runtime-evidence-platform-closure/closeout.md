# PM-UI-LAB-PERF-002 Runtime Evidence Platform Closure Closeout

## Scope

`WR-105` completes the bounded `PM-UI-LAB-PERF-002` implementation slice:
Editor Lab runtime evidence now records every PM002 no-gap target as either a
captured runtime artifact or a typed platform-impossible result backed by
capability probe metadata.

This closeout does not claim command/surface source-of-truth closure,
direct-manipulation UX closure, persistence/API/example ergonomics closure, or
final no-gap certification. Those remain owned by `PM-UI-LAB-PERF-003` through
`PM-UI-LAB-PERF-006`.

- `apps/runenwerk_editor/src/shell/editor_lab_evidence/mod.rs` module
  `editor_lab_evidence` adds typed evidence capabilities, capability support
  probes, captured/platform-impossible/failed results, platform-impossible
  artifact kinds, and `EditorLabEvidenceManifest::validate_no_gap_capabilities`.
- `apps/runenwerk_editor/src/shell/tests.rs` test
  `pm_ui_lab_perf_002_runtime_evidence_platform_closure` executes the app-owned
  Editor Lab shell/provider/project/apply/rollback paths and writes the PM002
  artifact bundle when `RUNENWERK_WRITE_PM_UI_LAB_PERF_002_EVIDENCE=1`.
- `domain/ui/ui_definition` remains behavior-free. Evidence execution,
  platform probes, artifact writing, runtime timing, and retained/native
  capability classification stay in `apps/runenwerk_editor`.

## Acceptance Evidence

- Descriptor-only evidence remains rejected by manifest validation.
- PM002 no-gap validation rejects missing capability results and
  platform-impossible results without typed probe metadata.
- Captured capability results require concrete runtime artifacts, reproduction
  commands, backend/environment metadata, and diagnostics.
- Platform-impossible capability results require typed probe metadata and a
  platform-impossible report artifact.
- PM006 unsupported checks are no longer sufficient for PM002 closeout; they
  are replaced by typed capability results.
- App-owned runtime evidence remains isolated to `apps/runenwerk_editor`.

# PM-EDITOR-UX-006 Shell And Product Pattern Polish Closeout

> Terminology note: Current name: UX Lab Scenarios; historical name: Story Lab.

## Summary

`PM-EDITOR-UX-006` / `WR-116` completes a runtime-proven reusable editor shell
product-pattern slice. The Editor UX Story Lab now has a product-pattern story
covering inspector, palette, diagnostics, preview, table, tree, tab,
toolbar/status, split, dock, state, focus, keyboard, overflow, accessibility,
diagnostics, timing, and native or platform-impossible evidence.

The reusable pattern source truth remains in `domain/editor/editor_shell`.
`apps/runenwerk_editor` owns native Story Lab execution and manifest validation.
Generic UI crates remain generic and do not own editor product-pattern
semantics.

This closeout does not claim all registered visible surface migration, game UI
readiness, or final local-native no-gap certification. Those remain visible in
`PM-EDITOR-UX-007` through `PM-EDITOR-UX-009`.

## Implementation Summary

- `domain/editor/editor_shell/src/story_lab/product_patterns.rs` module
  `product_patterns` adds `EditorUxProductPatternEvidence`,
  `SHELL_PRODUCT_PATTERNS_STORY_ID`,
  `shell_product_pattern_fixture_root`,
  `shell_product_pattern_evidence`, and required state coverage for the PM006
  product-pattern story.
- `domain/editor/editor_shell/src/story_lab/story.rs` struct `EditorUxStory`
  adds optional `product_pattern_evidence` and
  `with_product_pattern_evidence`, keeping reusable pattern proof metadata in
  editor product Story Lab contracts.
- `domain/editor/editor_shell/src/story_lab/catalog.rs` function
  `default_editor_ux_stories` registers the Shell Product Patterns story and
  targets typed retained widgets for inspector, palette, diagnostics, preview,
  table, tree, tabs, toolbar/status, and split/dock evidence.
- `domain/editor/editor_shell/src/story_lab/workbench.rs` function
  `ui_designer_workbench_fixture_view_model` preserves the established
  "retained preview available" acceptance phrase used by shell workflow tests.
- `apps/runenwerk_editor/src/shell/providers/self_authoring.rs` function
  `workbench_pane_from_canvas` preserves the same live provider acceptance
  phrase for the UI Designer Workbench canvas path.
- `apps/runenwerk_editor/src/shell/editor_lab_evidence/mod.rs` enum
  `EditorLabEvidenceArtifactKind` adds `ProductPatternReport`.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/manifest.rs` struct
  `EditorUxEvidenceRun` adds typed `product_pattern_evidence`, and
  `EditorUxEvidenceManifest::validate` rejects product-pattern stories without
  `ProductPatternReport`, focus traversal, accessibility, diagnostics, timing,
  typed pattern/state/interaction/route evidence, and native or
  platform-impossible proof.
- `apps/runenwerk_editor/src/shell/editor_ux_story_lab/runner.rs` function
  `run_story` emits product-pattern artifacts and PM-EDITOR-UX-006 runtime
  proof.

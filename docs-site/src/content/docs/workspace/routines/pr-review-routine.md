# Pull Request Review Routine

## Use when

Use this routine to review a proposed patch.

## Authority files to read

Read changed files, owning docs, tests, `AGENTS.md`, `DEPENDENCY_RULES.md`, `DOMAIN_MAP.md`, and `programming-principles.md`.

## Working files to inspect

Inspect changed files, call sites, tests, docs, and affected public APIs.

## What to decide before editing

Decide whether the patch is acceptable, needs changes, or needs more evidence.

## Patch rules

Tie every finding to an exact file and function, module, or section.

## Manual validation checklist

Check correctness, owner fit, dependency direction, API usability, docs impact, validation evidence, and the seven principles.

## Evidence to report

Report recommendation, findings, files inspected, validation evidence, risks, and next action.

## Optional local helpers

Run focused tests, docs validation, or diff tools when available.

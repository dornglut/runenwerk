# Phase Completion Drift Check Routine

Use this routine after a completed phase or slice before starting the next one.

Read the accepted scope, changed files, tests, docs, planning records, and closeout/report evidence.

Check:

- promised work vs implemented work;
- code/docs drift;
- dependency or ownership drift;
- validation that ran or could not run;
- known gaps;
- next safe action.

Do not start new implementation in this routine.

Manual validation: inspect changed files, expected tests, docs impact, and planning impact.

Final report: completed scope, files changed, evidence inspected, validation status, drift found, gaps, next action.

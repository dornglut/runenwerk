# Implementation Routine

Use this routine for bounded implementation work.

Read `AGENTS.md`, `AI_GUIDE.md`, `ARCHITECTURE.md`, `DEPENDENCY_RULES.md`, `DOMAIN_MAP.md`, `TESTING.md`, and the owning design, ADR, planning, or issue context.

Before editing, decide the owner, scope, non-goals, validation expectation, and documentation impact.

Patch only the smallest coherent owned scope. Keep dependency direction legal. Update docs when behavior, public API, or ownership changes.

Manual validation:

- authority files inspected;
- target files inspected;
- dependency direction checked;
- public API impact checked;
- tests to run named;
- command validation status reported.

Final report: files changed, exact functions/modules/sections, validation, risks, next step.

# Public API Review Routine

Use this routine to review whether a public crate or module is easy to discover, use, and compose correctly.

Read public exports, README files, usage docs, examples, tests, and owning design or crate docs.

Check names, constructors, normal workflow ergonomics, diagnostics, examples, and docs/test agreement.

Report findings with severity, file, function or module, issue, why it matters, recommended change, validation status, and risks.

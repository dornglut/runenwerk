# Complete Merge Readiness Gate

This workflow is superseded by [Engineering Workflow](engineering-workflow.md).

Historical reports may continue linking this path. It does not authorize new work.

The durable principle retained from this document is:

```text
a pull request is merge-ready only when its scope, acceptance criteria, ownership impact, required tests and documentation, reviewed-head baseline, CI, unresolved findings, and post-merge truth are known
```

The replacement workflow expresses this as one pull-request contract instead of repeating investigation, design, lifecycle, principle, decomposition, and reporting checklists.

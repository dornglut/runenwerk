# Docs Cleanup Task

Routine: `docs-site/src/content/docs/workspace/routines/docs-refactor-routine.md`

Use this card for documentation moves, pruning, merging, link repair, root-summary alignment, or rewrites.

Do not hide authority, lifecycle, planning, design, evidence, or implementation-scope changes as cleanup. Use the complete investigation gate when cleanup may change meaning or ownership.

Report files changed, links updated, evidence classes used, complete investigation/design gate status where applicable, merge-readiness status when relevant, validation status, and unresolved drift.
